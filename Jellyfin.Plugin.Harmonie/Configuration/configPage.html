Optional. Used as a last-resort fallback if a track can't be matched by tags. Translate the harmonie-side path prefix to the Jellyfin-side prefix. One per line, format: /harmonie/path => /jellyfin/path.

Smart playlists

The plugin watches for three kinds of playlist names. Add tracks to one and the rest fills in automatically.
  • [RADIO] — track radio. The first N tracks (set by "Radio seed count" below) are the seeds; the rest is plugin-managed. Reorder a track into the top N to make it a seed; the first track is the strongest anchor.
  • [DRIFT] — drifting walk. Add a single track and harmonie generates tracks that gradually move away from it.
  • [MIX] — auto-generated from your Jellyfin listening history. You don't add tracks; the plugin does it on a schedule. Anything you add manually is wiped on the next refresh.
Append n=<count> to override length, e.g. [DRIFT n=50] Long mix. For [MIX]: append days=N, top, or drift to override the defaults below per playlist.

Playlist defaults

Applied when a playlist title doesn't override them. Initial values match harmonie's own API defaults.
For [RADIO] playlists: the first N tracks in the playlist are the seeds harmonie matches against. Anything below position N is plugin-managed and gets refreshed. Reorder a track into the top N to make it a seed. 1–20.
Number of tracks for [RADIO] playlists. 1–500.
Number of tracks for [DRIFT] playlists. 1–500.
Tracks per anchor before re-anchoring on the most recent pick. Larger values stay closer to the seed; smaller drifts faster. 1–100. Drift mode only.
Maximum BPM gap between consecutive tracks. Leave empty for no constraint (harmonie's default).
When on, consecutive tracks must share a Camelot-compatible key (same key, ±1 wheel position, or parallel mode). Strict — tracks without key info are excluded.

[MIX] defaults

Mix playlists seed themselves from your Jellyfin listening history. The settings below shape what becomes a seed.
Number of tracks in a mix playlist. 1–500.
How far back to look for seed tracks. 1 = today's listens. 7 = the week. 30 = your monthly taste profile.
Maximum number of seeds drawn from listening history. 5–15 is the sweet spot — too few clones one track, too many blurs the average. 1–100.
Off (default): "today's mix" — seeds are the most recently played tracks in the window. On: "your sound" — seeds are the highest play-count tracks. Override per playlist with [MIX top].
Off (default): a "more like this" mix grouped around the centroid. On: a stretch mix that walks away from the seed for discovery. Override per playlist with [MIX drift].

[STYLE] cluster playlists

Optional. When enabled, the daily refresh task creates and maintains a fixed number of playlists per Jellyfin user, one per top style derived from that user's listening history. Each playlist is named [STYLE] Genre · Style. As tastes shift the playlists rename and re-fill themselves — no churn, no orphans.
Number of [STYLE] playlists maintained per user. Reducing this later removes the excess. 0–10.
How far back to look when computing each user's top styles. 1–365.
Number of tracks in each [STYLE] playlist. 1–500.
